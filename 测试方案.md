# 最大上下文子图 (MCS) 引擎 —— 验证测试计划

本测试计划针对 **MCS Phase 1（领域知识图谱）** 进行分阶段验证。Phase 1 的输入假设是**教科书 / 整理过的知识文档**，不考虑时序冲突，不依赖事件层与版本链。

涉及矛盾处理、版本时序、GC 等能力的用例统一标注 **[Phase 2]**，本期不验收。

---

## 1. 测试对象与核心不变量 (Invariants)

测试将监控以下核心机制的自洽性，与 `MCS技术方案.md` 严格对齐：

* **物理容量硬约束 (T_max):** 任意节点的一跳邻域 Token 总和 ≤ T，绝不突破。
* **单一无类型邻接边:** 全图只有一种无标签邻接边；**方向不在摄入时判定，仅在社区合并时涌现**（§4.4）。
* **存在性判定语义:** 新概念放置时，由 LLM 阅读锚点邻域子图，回答"该概念是否已经以某个节点 X 的形式存在于此"（§4.2），**不做"在上 / 在旁 / 在下"的三向方向判定**。
* **惰性社区合并:** 邻居重合度高时，在信息最全的合并时机让 LLM 论证"谁当上位"，宁可不合不可错合（§4.4）。
* **出处可回溯:** 概念节点的 `sources` 字段在创建、合并、修订中保持完整；任何答案可定位到文档块级出处。
* **再次摄入幂等:** 相同 `(doc_id, chunk_id, content_hash)` 的输入第二次摄入时，图结构不变。
* **[Phase 2] 矛盾驱动的活跃图 GC:** 见 Phase 2 测试方案，本期不测。

## 2. 评测阶段划分与路线图

### 阶段一：主干功能与检索召回

冷启动测试，验证"找锚点 → 导航邻域 → LLM 判断是否已存在 → 并入或新建"主链路是否成立。

* **测试集选型:** **MultiHop-RAG**
    * *理由:* 一次建图、多 query，贴合 MCS 大图多跳检索场景；指标为文档级 Hit@k / MAP@k / MRR@k。
* **输入流构造:**
    1. 语料文档逐篇摄入，构建共享持久图；
    2. 每个新概念走完整放置流程，禁止任何"摄入时方向判定"。
* **核心指标:**
    * **文档级检索召回:** query() 返回节点经 source_tracking 映射回文档，与 gold evidence 比对。
    * **一跳扇出达标率:** 全图无节点突破 T_max。

### 阶段二：深层多跳与拓扑演化压测

主干 Recall 达标后转入"图自组织生长"深度压测。

* **测试集选型:** **2WikiMultihopQA / MuSiQue**
    * *理由:* 推理链条更深（3-4 跳），构型干净，能压测降扇出、合并涌现方向之后的拓扑健康度。
* **核心指标:**
    * **Multi-hop Routing Success Rate:** LLM 沿无类型邻接边能否正确"穿透"到叶子节点而不卡在桥/枢纽上。
    * **中转角色退化率:** 临时合成枢纽在后续真概念插入时能否被正确吸收/替换（§2.3）。

### 阶段三：鲁棒性与同名消歧

现实噪声下的鲁棒性，**不**涉及矛盾 / 时序（Phase 1 范围内）。

* **测试集选型:** **GRS-QA (2024 真版)**
    * *理由:* 提供被正向/负向扰动过的逻辑推理图，可直接映射；可构造跨领域同名概念。
* **核心指标:**
    * **惰性合并准确率:** 两块原本不相连的分支因为共享底层子节点引发"共现重连"后，系统通过邻居重合度触发 LLM 二次校准合并的准确率。
    * **同名消歧率:** 跨领域同名概念（科技苹果 vs 水果苹果），在没有共同连线时能否依靠拓扑天然语义隔离（§4.5"测不出的那种，刚好是可以不管的那种"）。

### **[Phase 2]** 阶段四：矛盾处理与时序鲁棒性

依赖事件层、版本链、置信度、GC，**Phase 1 不验收**。

* **测试集选型:** MultiHop-RAG（时序新闻）+ 矛盾事件注入
* **核心指标:** 矛盾驱动 GC 软隔离有效性、版本时序召回准确率、撤回 / 纠正路径正确性

## 4. 评测执行与终止条件

1. **准入条件:** 内存图结构（无类型邻接表）+ Source 出处追踪 + 别名词典核心代码封装完毕；LLM 的"概念是否已存在"和"枢纽决策"两个核心 Prompt 调优通过。

2. **暂停条件:** 阶段一测试中若文档级 Hit@10 连续 50 个案例低于 30%，说明"找锚点 + 判断是否已存在"主链路存在严重问题，必须暂停并重新对齐放置语义。

3. **Phase 1 通过判定（同时满足）:**
    * 阶段一文档级 Hit@10 ≥ 0.7（有重排）；
    * 阶段三 同名消歧错误率 < 5%；
    * 全过程一跳扇出零次突破 T_max；
    * 再次摄入幂等性 100%（同输入零增长）；
    * 答案出处回溯命中率 ≥ 95%（带 sources 的节点被引用时正确给出 doc_id）。

4. **Phase 2 启动条件:** Phase 1 通过判定全部满足后，方可开始事件层、版本链、置信度、GC、矛盾仲裁等插件的设计与验证。

## 5. 已知限制（不在 Phase 1 验收范围内）

- **同名异义查询消歧**：词法 `lookup` 命中多个同名节点时直接合并为种子集合，由 `_traverse` 方向判断自然过滤；不做主动消歧。教科书场景下通过节点 `name` 字段去歧义化（"苹果(科技公司)"）+ 共现概念间接缓解。详见 `architecture.md` §11。
- **存储增量写入**：每次 ingest 后需手动 `save()`，不订阅图变更事件。
- **LLM 调用缓存**：无，串行调用。
- 上述限制不阻塞 Phase 1 通过判定。
