# MCS — LLM 原生数据结构

MCS 把零散文本组织成**自组织的语义知识图**，并维持一条硬不变量：任意节点的邻域永远放得进一个 LLM 上下文窗口。图会自己"长"和"缩"——不需要预定义本体，不会因为数据量增长而溢出。

```python
from mcs.presets import create_mcs

mcs = create_mcs(llm="deepseek", db_path="mcs.db")

mcs.ingest("深度学习是机器学习的一个子领域，使用多层神经网络学习数据表示。")
mcs.ingest("卷积神经网络是一种专门处理网格状数据的深度学习模型。")

nodes = mcs.query("什么是深度学习？")
for n in nodes:
    print(n.name, "—", n.content[:80])
```

> MCS 返回的是相关节点集合（`List[Node]`），不是自然语言答案。它专注于记忆本身，把合成答案、多轮对话留给上层（RAG / Agent / Chatbot）。

## 为什么需要一种新的数据结构

### 本体论套不住真实世界

传统知识图谱依赖预定义的本体——先设计类和关系，再把事实往里填。这对 80% 的问题有效，但剩下的 20% 永远对不上：真实世界不会按你的本体长，领域交叉、概念漂移、例外情况会持续出现。

MCS 的做法：**去掉本体论骨架，保留图谱的推理力**。图的结构由 LLM 语义归纳自动生长，不需要人预先定义"什么是类""什么关系合法"。图谱 + 事实推理是有用的东西，本体论不是。

### 数据量超过一个上下文窗口就崩

Agentic 记忆系统有一个根本矛盾：LLM 的推理能力来自"看到全部相关信息"，但上下文窗口有限。当记忆量超过窗口容量，性能就会断崖式下降——不是渐进退化，是结构性失效。

MCS 的做法：**硬不变量保证任意节点的邻域 ≤ T**。无论图增长到多大，LLM 永远能一次性读完任意节点的局部视野来做判断。溢出由语义裂变自动收敛，不是截断、不是丢弃，是对知识的重组。

### 树形目录描述不了复杂关系

文档 / 笔记系统的层次化目录是树结构。但知识天然是图——一个概念同时关联多个领域，一条事实涉及多个实体。用树来装图，要么信息散落各处找不到，要么强行归类扭曲语义。

MCS 的做法：**图是原生结构**。概念间的多对多关联用双向边直接表达，一个节点可以属于多个社区（重叠），导航沿语义边游走而非沿目录下钻。

## 核心机制

### 有界保证

MCS 维持一条硬不变量：

> **任意节点的活跃双向视图（关系边 + 层级邻居），渲染 token ≤ T**

这是导航、归纳、查询的共同地基。保证 LLM 无论在哪个节点、图增长到多大，都能一次性读完局部视野做语义判断。

### 语义裂变

当某节点的邻域即将超出 T，系统自动触发**聚类裂变**——不是截断，是对知识的重组：

```
膨胀              分裂                收敛
┌──────────┐     ┌─────┐ ┌─────┐    ┌─────┐ ┌─────┐
│  Hub     │     │ Grp │ │ Grp │    │ Grp │ │ Grp │
│ ·a  ·b  │ ──▶ │·a·b │ │·c·d │ ──▶ │·a·b │ │·c·d │
│ ·c  ·d  │     │  ·c │ │  ·e │    │  ·c │ │  ·e │
│ ·e  ·f  │     │     │ │  ·f │    │     │ │  ·f │
└──────────┘     └─────┘ └─────┘    └──┬──┘ └──┬──┘
  超出 T!         语义聚类              ┌─┴──────┘
                  (LLM 归纳)           │ 中心节点 │
                                       │ →GrpA   │
                                       │ →GrpB   │
                                       └─────────┘
                                        扇出收敛 ✓
```

裂变由 LLM 语义归纳驱动（`decide_hub`），不是图聚类算法。重组有三种方式：
- **合并同义**：旧的同义概念合并为一
- **找到关键概念**：识别社区中的关键概念，其余概念以它为组织中心
- **概括新概念**：无现成关键概念时，概括成新概念并关联旧概念

禁止空洞聚合标签（如"信息碎片集合"）——每个 hub 都必须是语义上有意义的概念。

### 写入管线

```
input: text
① 前置插件链 (可选)
② 关联节点定位 ◄── 复用读流程
③ 概念提取 (LLM)
④ 关系判定 (LLM) → DecisionList
⑤ 图更新 (无 LLM)
⑥ 压缩判定插件链 (条件触发 → 裂变)
⑦ 自动落盘 (SQLite 增量持久化)
OUTPUT: 图状态更新 + 已持久化
```

写入时先"读"图找锚点（读写对称），新概念自动挂入图、触发守门检查、必要时裂变收敛。

### 查询管线

```
input: query
① 前置插件链 (可选)
② 种子定位 (jieba 切词 + 字面匹配 → embedding 兜底)
③ 子图扩展 (BFS，LLM 选关系边 + 端点补入)
④ 后处理 (重排 / 裁剪)
⑤ 返回 Subgraph (nodes + 选中关系边)
```

种子定位以关键词为主力，不依赖 embedding 相似度。子图扩展沿语义边 BFS 游走，每步 LLM 选择相关关系边，自然收敛到与查询相关的子图。

## 特性

| 特性 | 说明 |
|------|------|
| **向量无关** | 不依赖 embedding / 向量检索，LLM 直接读子图做判断 |
| **有界保证** | 硬不变量确保任意节点邻域 ≤ T，数据量增长不退化 |
| **语义自组织** | 裂变由 LLM 归纳驱动，图自动生长和收敛 |
| **多 LLM 后端** | DeepSeek（默认）/ Claude / Ollama 本地，读写可用不同模型 |
| **插件体系** | 12 种插件类型，管线各段可插拔替换 |
| **双关系模式** | `property_graph`（默认，带 label 事实边）/ `attribute_node`（属性节点 + 无类型关联边） |
| **增量持久化** | SQLite 自动落盘，重启可续 |
| **零配置起步** | Mock 模式无需 API key，5 行代码跑通 |

## 评测

MultiHop-RAG 文档级多跳检索评测，608 篇文档、10,639 节点图：

| 配置 | 查询数 | Hit@10 | Recall@10 |
|------|--------|--------|-----------|
| DeepSeek, T=16k, LLM 重排 | 200 | **0.770** | 0.484 |
| GLM-5.1, T=16k, LLM 重排 | 200 | **0.765** | 0.487 |
| DeepSeek, T=32k, 词法重排 | 1000 | 0.649 | 0.438 |

关键发现：
- **建图质量主导检索效果，查询侧模型可替换**——DeepSeek 建图、GLM 查询，Hit@10 基本持平（0.765 vs 0.770）
- **相关性重排是关键**——从 ~0.16 提升到 ~0.73（词法重排即可）
- **不变量验证全通过**——608 篇文档建图后，fanout 口径 0 违反

详见 [bench/README.md](bench/README.md) 和 [评测报告](bench/multihop_rag/reports/index.md)。

## 快速开始

### 安装

```bash
git clone https://github.com/<your-org>/mcs.git
cd mcs
python -m venv .venv
.venv\Scripts\activate          # Windows
# source .venv/bin/activate     # macOS / Linux
pip install -e ".[dev]"
```

### 零配置跑通

```bash
python examples/basic_usage.py          # Mock 模式，无需 API key
```

### 接入真实 LLM

```python
from mcs.presets import create_mcs

# DeepSeek（默认）
mcs = create_mcs(
    llm="deepseek",
    db_path="mcs.db",
    plugin_configs={"deepseek_llm": {"api_key": "sk-..."}}
)

# Claude
mcs = create_mcs(
    llm="claude",
    db_path="mcs.db",
    plugin_configs={
        "claude_llm": {
            "auth_token": "your-token",
            "model": "claude-3-5-sonnet-latest",
            "base_url": "https://api.anthropic.com",
        }
    }
)

# Ollama 本地（零 token 成本）
mcs = create_mcs(
    llm="ollama",
    db_path="mcs.db",
    plugin_configs={
        "ollama_llm": {
            "base_url": "http://localhost:11434/v1",
            "model": "qwen3.5:9b",
        }
    }
)
```

### 评测复现

```bash
python -m bench.multihop_rag --llm deepseek --corpus-subset 200 --output ./mh_out
```

## 适用场景

MCS 最适合**已能被人类整理成可教结构的领域**——物理、工程、各类有教科书/本体的学科。核心假设是知识有足够的局部性：回答一个问题所需的概念，在图里彼此靠近，几跳语义游走就能连到一起。

典型用法：
- **RAG 的记忆层**：替代向量检索，返回结构化的相关子图
- **Agent 的长期记忆**：任务经验自动组织成图，跨会话持久化
- **领域知识库**：论文/文档自动入图，语义关联可导航
